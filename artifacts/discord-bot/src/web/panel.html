📖 How the Bot Works

A complete reference for everything the bot does — automatic protections running in the background, manual commands, the appeal system, and the web panel.

🤖 Automatic Protections (Always Running)

These run 24/7 with no staff involvement. The bot watches every message and server event.

⚡ Anti-Spam
If a user sends 5 or more messages within 5 seconds, the bot immediately:
1. Deletes the triggering message
2. Bulk-deletes their recent spam messages (up to 20 fetched, recent filtered)
3. Times the user out for 5 minutes
4. Logs the event to the server log channel

Whitelisted owners are fully exempt. The bot resets the spam counter after triggering.
📢 Mass Mention Detection
If a single message contains 3 or more user/role mentions:
• First offence — message deleted, warning logged
• Second offence — message deleted + user kicked from server
Strike counter resets after a kick. Whitelisted owners are exempt.
🔗 Link Filter
All links are blocked unless the sender has one of the four exempt roles. Specific rules:
• Discord invite links — always deleted for everyone, including exempt roles
• GIF hosting sites — always allowed: Tenor, Giphy, Gfycat, Imgur, Klipy
• Images and file attachments sent without any text — always allowed (not treated as a link)
• Masked links — even if hidden behind text like [click here](url), GIF hosts are still blocked if masked
• All other URLs — deleted and logged

The four exempt role IDs bypass all link filtering (Discord invite links still blocked).
🤬 Profanity Filter
Monitors all message text for serious slurs and explicit words. Mild words like wtf, shit, ass, damn are intentionally allowed.

When a blocked word is detected:
1. Message is deleted
2. User gets a DM explaining the deletion and how many strikes remain
3. After 4 strikes, the user receives a formal warning and their counter resets

The filter handles character substitutions (4 → a, @ → a, 1 → i, $ → s, etc.) and repeated letters to catch bypass attempts. Only applies to message text — images and GIFs are never flagged.
🤖 Unauthorized Bot Detection
When any bot joins the server, the bot checks if it is on the whitelist (set via /whitelist).
• If not whitelisted → bot is kicked, and the Discord user who added it gets a DM notification
• If whitelisted → bot stays, event is logged as allowed

This prevents server raids via malicious bots being added.
🛡️ Dangerous Role Creation
When a new role is created with Administrator permission:
• Audit logs are checked to find who created it
• If the creator is not the server owner or a whitelisted owner → role is immediately deleted and logged
• If the creator is the owner or whitelisted → allowed, but still logged as a warning

This prevents privilege escalation attacks.
💣 Mass Channel Deletion (Anti-Nuke)
Tracks channel deletions per user within a 30-second window.
• If someone deletes 4 or more channels in 30 seconds → all their roles are stripped immediately
• This neutralizes the threat instantly without banning (in case it was accidental)
• Stripped roles are saved so an admin can restore them manually

Special case — Log Channel: If the dedicated server log channel is deleted:
1. The deleter is permanently banned (DM sent with appeal link)
2. A new log channel is automatically recreated
3. All whitelisted owners and the bot get access to the new channel
4. A full incident report is posted in the new channel
🪝 Webhook Deletion
Any new webhook created in the server is automatically deleted unless it was created by the server owner or a whitelisted owner.
Webhooks are a common attack vector for raiding — this removes them before they can be used.
📝 Message Deletion Tracking (Snipe)
Every deleted message in the server is recorded internally — including the content, the author, and who deleted it (if it was deleted by someone else, that is logged too).
Staff can use /snipe @user to view the last 10 deleted messages from any user.

⚡ Manual Commands — Moderation

/ban <user> [reason]
Bans a user from the server. Requires Ban Members permission.
• Checks the executor has permission and the target is not a higher role
• DMs the banned user their ban reason and the appeal server link
• Logs the action to the server log channel
/kick <user> [reason]
Kicks a user. Requires Kick Members permission.
• Role hierarchy is checked before kicking
• DMs the user with the reason
• Logged to the log channel
/timeout <user> <minutes> [reason]
Times out a user for the specified number of minutes (max ~28 days).
• Requires Moderate Members permission
• DMs the user with the duration and reason
• Logged
/warn <user> [reason]
Issues a formal warning. Warns are tracked permanently per guild.
• Warn 1 → 24-hour timeout
• Warn 2 → 48-hour timeout
• Warn 3+ → ban (with appeal DM)
• Each warn is logged and viewable via /warns
/warns <user>
Lists all recorded warnings for a user, showing the reason and timestamp of each one.
/clearwarns <user>
Clears all warnings for a user. Requires ban permissions. Logged.
/purge <amount> [@user]
Bulk-deletes up to the specified number of recent messages in the channel.
If a user is specified, only their messages are deleted. Requires Manage Messages permission.
/lockdown <lock|unlock> [reason]
Locks the current channel (removes Send Messages from @everyone) or unlocks it. Logged with reason.
/snipe <user>
Shows the last 10 deleted messages from the specified user in the server. Includes content, channel, and timestamp.

🔧 Manual Commands — Utility & Config

/userinfo [@user]
Shows detailed info about a user: join date, account age, roles, and total warn count. Defaults to the command user if no target given.
/serverinfo
Shows server statistics: member count, boost level, channel count, creation date, and server owner.
/roblox
Fetches live Roblox game stats for Survive Events for Brainrots: current players online, total visits, favorites, like/dislike counts — with direct buttons to the game page.
/prefix <prefix>
Sets a custom text command prefix for the server (e.g. !). After setting, all slash commands are also usable as text commands (e.g. !ban @user reason). Server owner only.
/whitelist <botId>
Adds a bot ID to the whitelist so it won't be auto-kicked when joining. Server owner only.
/whitelistowner <user>
Adds a user to the whitelist. Whitelisted users bypass all auto-protections (spam, link filter, profanity, anti-nuke checks). Server owner only.
/unwhitelistowner <user>
Removes a user from the whitelist. They are now subject to all auto-protections again.
/usercreate <user> <username> <password>
Creates a web panel account for a Discord user. Server owner only.
• Hashes the password and stores the account
• Generates a QR login token and QR code image
• DMs the user their credentials, QR image, and panel link

📩 Ban Appeal System

Runs entirely via Discord DMs — users interact with the bot directly in their private messages.

How Appeals Start
The appeal system posts a message in the appeal server with two buttons: Server Ban Appeal and In-Game Ban Appeal.
When a user clicks, the bot checks if they are actually banned (for server appeals), then DMs them to begin the questionnaire.
Server Ban Appeal (12 Questions)
The bot asks 12 questions one at a time via DM:
username confirmation → ban date → known reason → fairness explanation → rule compliance → prior warns → conflicts → other parties involved → evidence → their perspective → what they'd do differently → anything else

The bot fetches the actual ban reason from the main server and includes it in the appeal embed.
In-Game Ban Appeal
Starts with asking why they were banned. If the answer mentions anti-cheat keywords (EAC, VAC, BattlEye, etc.) → the bot flags it as an anti-cheat ban, does not accept normal appeals, and asks for a video recording instead.
For normal game bans → asks for a screenshot of the ban message, then 6 more questions.
AI Detection
After the user completes all questions, their combined answers are analyzed by the built-in AI detector before submission.

The detector checks for:
• Known AI phrases: "I sincerely apologize", "Going forward", "I take full responsibility", "Furthermore", etc.
• Overly formal transition words (Moreover, Consequently, In conclusion…)
• Uniform sentence lengths (AI writes consistently; humans vary)
• Multiple structured paragraphs
• Overly formal vocabulary (sincerely, humbly, wholeheartedly, utmost…)
• Human markers that reduce the score: casual slang, lowercase i, short answers, contractions

If AI is detected → appeal rejected, user gets a DM explaining why and can try again. If it passes → appeal is submitted to the appeal channel with an AI scan result field.
Staff Review
Approved appeals are posted in the appeal channel with Approve/Deny buttons visible to staff with the Approver role.
• Clicking either button opens a modal asking for a written reason
• The reason is sent to the user via DM only (not posted publicly)
• On approval of a server ban → user is automatically unbanned + sent a server invite
• On approval of a game ban → user is told to wait 1–2 days
• All appeal outcomes are stored and visible in the web panel Appeals section

🌐 Web Panel

Login
Two login methods:
• Password — enter username and password (bcrypt-hashed, never stored in plain inside the session)
• QR Token — paste the token from your bot DM to log in without a username or password

Sessions last 7 days via a secure HTTP-only JWT cookie.

Moderation (Web)
Staff can take moderation actions on Discord users by entering their User ID:
• Timeout — set duration in minutes (up to 28 days)
• Kick — removes from server, no ban
• Ban — permanent ban, DM sent to user with appeal link, appeal entry created in the system
• Warn — adds a warn entry, DMs the user
All web actions are logged to the server log channel with the staff member's username.
Logs
Displays the last 200 moderation actions recorded by the bot — including automatic protections (spam timeouts, link deletions, anti-nuke actions) and manual commands. Each log shows the action title, description, and timestamp.
Appeals (Web)
Shows all submitted appeals with filtering by status (pending / approved / denied). Displays the user, appeal type (server/game), status, submission date, and who reviewed it. Appeals are also handled via Discord buttons in the appeal channel — the web panel is read-only for this.
Settings
• Theme — choose panel color: Default (purple), Red, Blue, Green, Light
• Language — save your preferred language preference (20+ options)
• Change Password — enter new password → bot DMs you a confirm button → click confirm in Discord → password updates instantly
• Account Info — shows your username, Discord ID, and current access level
Shift User
Transfers your panel account to a different Discord user (Administrator+ only):
• If the target has no existing account → bot DMs them with Accept/Decline buttons. On accept, an account is created with their Discord username and a random password. QR code is included in the DM.
• If the target already has an account → this account is immediately reassigned to them. They are DM'd about the transfer.
• Server owners and whitelisted users are always notified of any shift.
Accounts (Owner / Whitelist only)
Lists every panel account with:
• Username
• Linked Discord ID
• The password (stored at creation time)
• Creation date
Accounts can be deleted from this page. You cannot delete your own account.

📋 Logging

What Gets Logged
Every significant action is logged to the designated server log channel and stored in the bot's data file for the web panel:
• All moderation actions (ban, kick, timeout, warn, purge, lockdown)
• Anti-spam triggers
• Mass mention warnings and kicks
• Link and profanity deletions
• Anti-nuke events (role stripped, channel deletion spree)
• Dangerous role creation
• Unauthorized bot kicks
• Webhook deletions
• Log channel recreation events
• Web panel moderation actions (with staff username)
Log Channel Protection
The log channel is set to a specific channel ID. If that channel is deleted:
1. The deleter is immediately banned
2. A new log channel is created with restricted permissions (only Administrators and the bot can view it)
3. A full incident report is posted — who deleted it, old channel ID, new channel ID
4. The new channel ID is saved so logging continues seamlessly
Data Storage
All persistent data is stored in data/botdata.json:
• Warns per guild per user
• Whitelisted bots and owners
• Guild prefixes
• Deleted message history (for /snipe)
• Channel deletion trackers (for anti-nuke)
• Stripped role backups
• All appeals (last 500)
• Moderation logs (last 200)
• Web panel user accounts (with hashed passwords and QR tokens)
• Pending password changes and shift confirmations
